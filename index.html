Q1. What is Java?

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. It is widely used for building enterprise applications, mobile applications (Android), and large-scale systems. Java is platform-independent due to the "Write Once, Run Anywhere" (WORA) capability, enabled by the Java Virtual Machine (JVM).

Q2. What are the features of Java?

Java offers several key features:

  • Platform Independence: Java programs run on any operating system with JVM.
  • Object-Oriented: Java follows OOP principles like encapsulation, inheritance, and polymorphism.
  • Security: Java provides secure execution using bytecode verification and sandboxing.
  • Robust: Java has strong memory management, exception handling, and type-checking mechanisms.
  • Multithreading: Java supports concurrent execution using threads.
  • Garbage Collection: Automatic memory management via JVM.

Q3. What is JVM and why is it important?

JVM (Java Virtual Machine) is a part of Java Runtime Environment (JRE) that converts Java bytecode into machine code. JVM enables Java's platform independence and provides memory management, security, and garbage collection.

Q4. What is the difference between JDK, JRE, and JVM?

  • JDK (Java Development Kit): Contains tools to develop and compile Java programs, including JRE and development utilities.
  • JRE (Java Runtime Environment): Provides runtime libraries and JVM to run Java applications.
  • JVM (Java Virtual Machine): Converts bytecode into machine code and executes Java programs.

Q5. What is the use of the public static void main(String[] args) method?

This method is the entry point of a Java program. JVM looks for this method to start execution. Explanation of keywords:

  • public: Accessible from anywhere.
  • static: Can be called without creating an object.
  • void: Does not return any value.
  • main: Special method name recognized by JVM.
  • String[] args: Array for command-line arguments.

Q6. Explain Object-Oriented Programming in Java.

Object-Oriented Programming (OOP) is a programming paradigm based on objects. Java follows four main OOP principles:

  • Encapsulation: Hiding data using access modifiers.
  • Inheritance: Allowing a class to derive properties from another class.
  • Polymorphism: Methods behaving differently based on input.
  • Abstraction: Hiding implementation details and exposing only necessary functionalities.

Q7. What is inheritance in Java?

Inheritance is a mechanism that allows a child class to acquire properties and behaviors of a parent class using the extends keyword. Example: 

class Animal {
    void makeSound() {
        System.out.println("Animal makes a sound");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("Dog barks");
    }
}

Q8. What is polymorphism in Java?

Polymorphism allows an object to take multiple forms. It is achieved via:

  • Method Overloading: Same method name with different parameters.
  • Method Overriding: Redefining a method in a subclass.
Example of Method Overloading: 
class MathUtils {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
}

Q9. What is encapsulation?

Encapsulation is the technique of restricting direct access to class fields and methods while providing controlled access via getters and setters. Example: 

class Person {
    private String name;
    public String getName() { return name; }
    public void setName(String newName) { this.name = newName; }
}

Q10. What is an interface in Java?

An interface is a reference type that defines a contract for classes to implement. Interfaces contain abstract methods that must be implemented by concrete classes. Example: 

interface Animal {
    void makeSound();
}
class Dog implements Animal {
    public void makeSound() {
        System.out.println("Dog barks");
    }
}
